your48 System Architecture

CIVIC INTELLIGENCE PLATFORM • CHICAGO 48TH WARD

Frontend Layer

  • Next.js 15 with App Router for SSR
  • Tailwind CSS + shadcn/ui components
  • Mapbox GL for precinct boundary overlay
  • Fuse.js for client-side fuzzy search
  • React Hook Form + Zod validation
  • Mobile-first responsive design

API Layer

  • Vercel Serverless Functions (Edge runtime)
  • RESTful API routes for all entities
  • Authentication via environment variables
  • Rate limiting (100 req/min per IP)
  • CORS configured for your48.org only
  • Input sanitization on all endpoints

Data Layer

  • Airtable: Tickets, initiatives, events
  • Static JSON: Directory data (edge-cached)
  • Vercel Cron: Daily/weekly sync jobs
  • Chicago Data Portal API (50K req/day)
  • Board of Elections GeoJSON
  • Total data: ~17MB (well within limits)

Automation

  • Vercel Cron: Scheduled data syncs
  • Node.js scripts: Data normalization
  • Error handling: Retry with backoff
  • Logging: All sync operations logged
  • Fallback: Static data if API fails
  • Cost: $0 (Vercel free tier)

Processing

  • ROSTR Framework: AI orchestration
  • PAL: User intent compilation
  • NPAO: Task phase classification
  • Data validation: Schema enforcement
  • Geocoding: Address → coordinates
  • Precinct detection: Point-in-polygon

Security

  • HTTPS-only (TLS 1.3)
  • API keys in env vars (never client-side)
  • CSRF tokens on all forms
  • XSS prevention (input escaping)
  • Rate limiting prevents abuse
  • WCAG 2.1 AA accessibility

Data Flow: Search

  • 1. User types in search box
  • 2. Fuse.js queries local JSON cache
  • 3. Results render in <100ms
  • 4. Click → API fetches full record
  • 5. Detail page renders with map pin

Data Flow: Intake Form

  • 1. User fills form (address auto-detect precinct)
  • 2. Zod validates, photos upload to S3
  • 3. API creates Airtable record
  • 4. Confirmation email sent
  • 5. Staff dashboard shows new ticket

Data Flow: Sync

  • 1. Vercel Cron triggers daily at 2 AM
  • 2. Script fetches from Data Portal API
  • 3. Data normalized, validated
  • 4. Airtable updated (upsert)
  • 5. Static JSON regenerated on Vercel
